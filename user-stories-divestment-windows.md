## Safeguard: divestment with/without penalty

| Requirement | User Story | Acceptance Criteria | Screenshot |
|---|---|---|---|
| Return investor principal on **the next** divestment window after divestment initiated | As an investor, I want my divestment principal returned on the next divestment window after I initiate the divestment, so that I have a clear and predictable timeline for receiving my funds. | GIVEN an investor has confirmed a divestment request by tapping "Lanjut" on the Konfirmasi page WHEN the system processes the divestment THEN the principal (pokok) must be returned to the investor's account on the 28th of the next divestment window Notes: 1. Divestment windows fall quarterly: 28 January, 28 April, 28 July, 28 October 2. The "next" window is determined by the date the divestment was initiated — e.g. if initiated on 5 March 2026, the next window is 28 April 2026; if initiated on 29 April 2026, the next window is 28 July 2026 3. If initiated on the 28th of a divestment month itself, funds settle on the 28th of the next divestment window (not the current one) 4. If the investment is < 12 months, the 1% early withdrawal penalty must be deducted from the principal before settlement 5. Keuntungan (returns) is excluded from the divestment — only pokok is returned 6. The system must send the investor a confirmation with the expected settlement date matching the "Dana akan diterima pada" date shown on the Konfirmasi page | N/A (backend process) |
| Allow investors to divest an asset with or without penalty based on investment duration | As an investor, I want to divest my asset from the Investment Detail Page, so that I can exit my position — penalty-free after 12 months, or early with a clear penalty disclosure if within 12 months. | GIVEN users are in the GGS Investment Detail Page AND the Investment Detail Page displays: asset value, keuntungan, investment start date, payout frequency, next payout date range, and info sections (Detail investasi, Daftar penerima, Informasi penarikan) WHEN they tap "Ajukan Penarikan" THEN the system checks the investment duration: **Scenario A — Investment >= 12 months (no penalty):** - User is taken directly to the Konfirmasi page showing: - Product name (e.g. "Grassroots Growth Series") - "Growth Series Dynamic" with the withdrawal amount (pokok only) - "Biaya admin: Gratis" - "Total" showing the full withdrawal amount - Note: "Dana akan diterima pada [25-28] [month] [year]" - "Kembali" and "Lanjut" buttons **Scenario B — Investment < 12 months (with penalty):** - A bottom sheet modal appears showing: "Penarikan akan dikenakan biaya 1% dari nilai aset" with explanation that the penalty applies because the investment hasn't reached 12 months, and "Kembali" / "Oke, Mengerti" buttons - WHEN user taps "Oke, Mengerti" THEN the modal is dismissed AND additional withdrawal info appears on the Investment Detail Page: "Penarikan paling lambat pada" with deadline date, "Biaya admin: Gratis" - WHEN the user proceeds to the Konfirmasi page THEN it shows: - Product name (e.g. "Grassroots Growth Series") - "Growth Series Dynamic" with the asset amount - "Biaya penarikan lebih awal (1%)" as a separate line item showing the deducted penalty (e.g. -Rp2.000.000) - "Biaya admin: Gratis" - "Total" showing the net amount after penalty deduction - Note: "Dana akan diterima pada [25-28] [month] [year]" - "Kembali" and "Lanjut" buttons Notes: 1. Only pokok (principal) is withdrawn — keuntungan (returns) is excluded ("Penarikan > exclude keuntungan") 2. Early withdrawal penalty is 1% of the asset value 3. The warning modal must be shown before the user can proceed — they must explicitly acknowledge the penalty by tapping "Oke, Mengerti" | Investment Detail Page, Bottom sheet modal (< 12 months), Page / Confirmation (both paths) |
